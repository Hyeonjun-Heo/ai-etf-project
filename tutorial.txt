3. main branch : 말 그대로 메인 브랜치로써, 최종적으로 에러가 나지 않은 파일을 업로드 하는 곳. 
4. develop branch : main 브랜치의 복사본. 대부분의 완성된 파일을 develop branch로 업로드 할 것이며, 최종적으로 디벨롭 브랜치에서 검토 후 
머지를 통해 main 브랜치로 보내게 됨. 특히 브랜치끼리 꼬여버리거나 하는 상황이 발생할 수 있기 때문에 메인 브랜치를 보호하기 위함임.

6. branch 이동하는 방법:
    (1) 브랜치 생성 :  git branch "생성할 브랜치 이름"
    (2) 브랜치 목록 : git branch '*'표시가 붙어있으면 현재 브랜치
    (3) 브랜치 전환 : git checkout "전환할 브랜치 이름"
    (4) 브랜치 생성 후 전환 :  git checkout -b "생성 후 전환할 브랜치 이름"
    (5) 브랜치 최신화 : git fetch -p
